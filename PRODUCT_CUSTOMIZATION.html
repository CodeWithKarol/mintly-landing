Understanding the Product Structure

The landing page displays products and features in several organized sections. Understanding this structure will make customization easier.

Main Sections on Your Landing Page

  • Hero Section - Main headline, CTA, and dashboard visuals
  • Bento Grid - Modern grid layout showing core benefits
  • How-It-Works - 3-step process timeline
  • USP Section - Unique Selling Proposition cards
  • Social Proof - Testimonials and Trust Logos
  • Pricing - Monthly/Yearly toggle with 3 tiers
  • FAQ - Accordion-style questions
  • Footer - Links, newsletter, and social media

File Organization

File Purpose What It Contains
index.html Page Structure All HTML elements and content
style.css Visual Styling Colors, fonts, layouts, animations
script.js Interactivity Video modal, pricing toggle, scroll animations

Editing Hero Section

The hero section is the first thing visitors see. It's crucial for making a strong first impression.

How to Edit Step-by-Step

1 Change the Main Headline

<h1 class="hero-heading"> Master Your Money with <span class="text-gradient">AI Intelligence</span> </h1>

To change: Replace the text. The text inside the <span> will have the gradient effect.

2 Update the Subheadline

<p class="hero-subheading"> Stop guessing. Start growing. Mintly automates your savings... </p>

3 Modify Buttons

<a href="#signup" class="btn btn-primary btn-lg"> Start Free Trial </a>

Change the text or the href to link to a different page.

Modifying Bento Grid (Features)

The features are displayed in a modern "Bento Grid" layout with large and small cards.

Structure of a Bento Card

<div class="bento-card large-card card-ai"> <div class="card-content"> <h3>AI-Powered Automation</h3> <p>Description here...</p> </div> <div class="card-visual"> <img src="..." alt="..." /> </div> </div>

Customization Tips

  • Large Cards: Use large-card class. These take up 2 columns on desktop.
  • Small Cards: Use small-card class. These take up 1 column.
  • Icons: Replace the SVG inside .icon-wrapper.
  • Images: Update the src attribute of the images in large cards.

Updating How-It-Works Steps

The 3-step process section shows users how to get started.

Structure of a Step

<div class="process-step"> <div class="step-content"> <div class="step-badge">Step 01</div> <h3>Connect your accounts</h3> <p>Description...</p> </div> <div class="step-visual">...</div> </div>
Note: The second step has the class reverse to alternate the layout (text on right, image on left).

Customizing Pricing Plans

The pricing section features a toggle for Monthly/Yearly billing.

Updating Prices

Prices are stored in data attributes to allow the toggle to work:

<span class="price-value" data-monthly="12" data-yearly="10"> 12 </span>
  • data-monthly: Price shown when "Monthly" is selected.
  • data-yearly: Price shown when "Yearly" is selected.
  • Inner Text: The default price (usually monthly).

Highlighting a Plan

Add the popular class to a pricing-card to make it stand out (larger size, different border).

Managing FAQ Section

The FAQ section uses an accordion style.

Adding a New Question

Copy an existing accordion-item block:

<div class="accordion-item"> <button class="accordion-trigger" aria-expanded="false"> <span class="accordion-title">Your Question Here?</span> <span class="accordion-icon">...</span> </button> <div class="accordion-content" aria-hidden="true"> <div class="accordion-body"> <p>Your answer here.</p> </div> </div> </div>

The JavaScript in script.js automatically handles the opening and closing logic for any new items you add.

Updating Video Modal

The "Watch Demo" button opens a modal with a video.

Changing the Video

Locate the #video-modal section at the bottom of index.html:

<video id="demo-video" controls width="100%"> <source src="YOUR_VIDEO_URL.mp4" type="video/mp4" /> </video>

Replace the src attribute with the URL of your video file.

Common Customization Tasks

Change Brand Colors

All colors are defined as CSS variables in style.css:

:root { --primary: #0D9488; --primary-dark: #0F766E; --secondary: #0F172A; /* ... */ }

Animations

To animate an element when it scrolls into view, add the class reveal-element to it in HTML. The JavaScript observer will automatically handle the fade-in effect.

Testing Your Changes

Always test after making modifications.

  • Responsiveness: Check on mobile, tablet, and desktop.
  • Interactions: Test the pricing toggle, FAQ accordion, and video modal.
  • Console: Check browser developer tools (F12) for any JavaScript errors.
